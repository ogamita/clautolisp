#+TITLE: autolisp-file-compat

This module hosts compatibility-audit tooling for AutoLISP file, stream, and
printer behavior.

Its purpose is to support:

- local compatibility scenarios against the current =clautolisp= implementation,
- cross-implementation comparisons between SBCL and CCL host behavior where that
  is relevant,
- later product-level comparisons against AutoCAD, BricsCAD, or other target
  environments.

The initial implementation provides:

- scenario and report data structures,
- a portable local runner for byte/text file round-trips,
- a builtin-oriented runner that executes selected AutoLISP file, pathname, and
  printer functions inside temporary workspaces,
- multi-step builtin execution with step bindings for stream and file-descriptor
  workflows,
- executable coverage for explicit `OPEN` encodings, missing-file open failures,
  and printer-to-`READ` roundtrips through temporary files,
- executable coverage for unsupported encoding failures and line-by-line
  read-back of file-targeted printer output,
- executable coverage for explicit-encoding append workflows and `PRINT`-style
  blank-line output,
- executable coverage for relative stream append and printer/read-back
  workflows resolved through explicit AutoLISP current-directory state,
- executable coverage for backslash-delimited relative subpaths and `PRIN1`
  compound-form roundtrips under explicit current-directory state,
- executable coverage for `VL-FILE-SIZE` and `VL-FILE-DELETE` on relative paths
  under explicit current-directory state,
- executable coverage for `VL-FILE-RENAME` on current-directory-relative paths,
  including backslash-delimited subpaths,
- executable coverage for negative current-directory rename behavior,
- executable coverage for `VL-FILE-COPY` on current-directory-relative paths,
- executable coverage for negative current-directory copy behavior,
- executable coverage for `VL-FILENAME-MKTEMP` under explicit current-directory
  state,
- byte, text, and line-oriented comparison helpers,
- artifact capture suitable for later report emission,
- declarative scenario files under =scenarios/=, including nested newline and
  encoding, path, mutation, printer, and stream scenario directories,
- a command-line driver under =tools/run-file-compat/=,
- machine-readable report emission in s-expression or JSON form, with aggregate
  summary counts,
- SBCL and CCL runner modes through the wrapper script,
- product-runner adapter modes for AutoCAD and BricsCAD through external runner
  templates.
- GitLab CI integration for the local SBCL and CCL modes, with machine-readable
  JSON artifacts stored under =clautolisp/.artifacts/= during CI runs.

The current builtin corpus contains 69 declarative scenarios and 140 checks,
all passing on both SBCL and CCL.

The current driver can also:

- collect scenario files recursively from scenario directories,
- filter runs by classification,
- filter runs by one or more scenario tags.

The =clautolisp= subproject =Makefile= provides convenience targets for:

- =make -C clautolisp run-file-compat-sbcl=
- =make -C clautolisp run-file-compat-ccl=
- =make -C clautolisp run-file-compat-autocad=
- =make -C clautolisp run-file-compat-bricscad=
- =make -C clautolisp audit-file-compat-autocad=
- =make -C clautolisp audit-file-compat-bricscad=
