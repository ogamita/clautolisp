* Purpose
~autolisp-runtime~ is the runtime-layer module for AutoLISP values and later
evaluation state.

Its first responsibility is to define the concrete Common Lisp
representation of AutoLISP runtime objects and the handoff from
~autolisp-reader~ objects to runtime values.

* Main Files
- ~PLAN.md~
- ~documentation/design.org~

* Status
The initial runtime object model now exists.

Current implementation scope:

- runtime wrapper structures for non-immediate AutoLISP values,
- symbol interning independent from Common Lisp packages,
- reader-object to runtime-value mapping for literal data,
- explicit namespaces, value/function cells, and dynamic frames,
- explicit host-entry semantics for current document and current namespace,
- dynamic propagation of the active evaluation context into `SUBR` execution,
- clarified `boundp` semantics, including the documented "undefined becomes
  nil-bound" behavior,
- a first evaluator slice covering literals, symbol lookup, `quote`, `setq`,
  `progn`, `if`, `cond`, `and`, `or`, `while`, `repeat`, `foreach`, `lambda`,
  `function`, `defun`, `defun-q`, and user function calls,
- compatibility-preserved function list definitions for the `defun-q` family,
- a first runtime `load` entry point that reads and evaluates a file in a
  chosen evaluation context,
- a first `*error*` hook path for top-level runtime failures together with a
  runtime catch-all error object used by the Visual LISP catch-all layer,
- conservative session-level `ERRNO` tracking,
- distinct runtime conditions for `exit` and `quit`,
- a first namespace-exit bridge for `vl-exit-with-error` and
  `vl-exit-with-value`,
- an initial `autolisp-runtime-error` condition used to report evaluator and
  wrapped host-Lisp failures without leaking raw Common Lisp errors directly.
