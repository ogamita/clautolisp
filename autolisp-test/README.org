* Purpose
~autolisp-test~ is the planned conformance-test subproject for ~clautolisp~.

Its role is analogous to Common Lisp ~ansi-tests~:

- define specification-oriented language tests,
- run those tests against a selected implementation version,
- record the selected host profile separately from the native operating system,
- produce a report that distinguishes passed tests, failed tests, expected failures, and unsupported areas.

This subproject is specifically for language-conformance testing.

It should not become a grab bag of:

- implementation-specific regression tests,
- host-integration smoke tests,
- UI tests,
- performance benchmarks.

Those belong elsewhere.

* Status
This subproject is currently in planning mode.

The directory structure and development plan exist, but the reusable harness and first conformance corpus have not yet been implemented.

* Planned Layout

The harness will hold the reusable runner, test registry, report generation, and host/version metadata support.

The tests tree will hold conformance tests grouped by language area, starting with:

- special forms,
- core functions shared in some form with Common Lisp,
- AutoLISP-only functions after the first projection pass.
