* Main Conclusions from the Source Corpus
- There is no single vendor-neutral AutoLISP / Visual LISP standard document comparable to the Common Lisp HyperSpec.
- Autodesk's documentation corpus is the closest thing to a normative reference for the core language.
- BricsCAD explicitly targets AutoLISP compatibility, but also extends the language/runtime significantly.
- Version changes matter, especially for:
  - Unicode behavior,
  - Visual LISP / ActiveX availability,
  - AutoCAD LT support,
  - BricsCAD VLE and optimizer features,
  - BricsCAD packaging, namespace, and vertical APIs.
- Any implementation that aims at compatibility needs dialect and version switches, not a single undifferentiated behavior model.
